# M3 黄金样本回归测试说明（基金属性表银行间 ID 匹配）

## 1. 目的

以**全脱敏虚构样本**为输入，对 M3 引擎（`server/app/engines/m3_interbank_id.py`）
建立回归基线：先人工按业务规格推导理论预期（本文件第 3 节），再由引擎实际运行
固化，防止"将错就错"。

## 2. 文件清单

| 文件 | 说明 |
|------|------|
| `make_samples_m3.py` | 样本构造脚本（全脱敏虚构数据） |
| `build_expected_m3.py` | 预期结果固化脚本 |
| `test_golden_m3.py` | 回归测试（纯脚本） |
| `samples/基金属性表_样本.xlsx` | 5 列 × 15 行（银行间ID 为数值/空值混合） |
| `samples/交易成员基本信息表_样本.csv` | 3 列 × 12 行，GBK 编码 |
| `expected/` | 预期三件套 + `expected_stats.json` + `expected_detail_df.pkl` |

## 3. 样本场景 → 理论预期对照表（人工推导）

### 3.1 逐行预期（基金属性表 15 行，顺序与输入一致，引擎不排序）

| 行(Excel) | 基金全称 | 原银行间ID | 成员表对应 | 匹配类型 | 颜色 | 更新后ID |
|---|---|---|---|---|---|---|
| 2 | 测试稳利1号 | 1001(数值) | 测试稳利1号/1001 | 精确匹配-无变化 | 蓝 BDD7EE | 1001 |
| 3 | 测试稳利2号 | 1002 | 测试稳利2号/1002 | 精确匹配-无变化 | 蓝 | 1002 |

| 7 | 测试进取6号 | 1006 | 测试进取6号/MB1006 | 精确匹配-有变动 | 绿 | MB1006 |
| 8 | 测试平衡7号 | （空） | 测试平衡7号/MB1007 | 精确匹配-有变动 | 绿 | MB1007 |

| 13 | 测试新发产品1号 | （空） | 无 | 未匹配 | 红 | （保持空） |
| 14 | 测试新发产品2号 | （空） | 无 | 未匹配 | 红 | （保持空） |
| 15 | 测试长青14号A类 | （空） | 无（成员表为"测试长青14号"） | 未匹配 | 红 | （保持空） |
| 16 | 测试恒远15号集合 | （空） | 无（成员表为"测试恒远15号"） | 未匹配 | 红 | （保持空） |

关键校验点（类型陷阱）：原ID `1001`（Excel 数值存储，pandas 读回 float64）
必须归一为字符串 `'1001'` 后再比较，方能与成员表字符串 ID `'1001'` 判等得到
「无变化」；若直接 `str(1001.0)` → `'1001.0'` 将误判为「有变动」，
本场景即防回归金丝雀。

### 3.2 预期统计

```json
{
    "总记录数": 15,
    "精确匹配-有变动": 5,
    "精确匹配-无变化": 4,
    "未匹配": 6
}
```

### 3.3 未匹配原因归类预期

| 基金全称 | 归类 |
|---|---|
| TR_测试现金管理1号 / TR_测试现金管理2号 | 内部简称（TR_ 前缀） |
| 测试长青14号A类 / 测试恒远15号集合 | 命名后缀不一致（与成员全称互为包含） |
| 测试新发产品1号 / 测试新发产品2号 | 未注册产品（无相近名称） |

### 3.4 输出三件套预期

1. `基金属性_精确匹配更新.xlsx`：5 列全保留，16 行（表头+15），
   行 2-5 蓝、行 6-10 绿、行 11-16 红，银行间ID 列值按 3.1「更新后ID」；
2. `精确匹配结果明细.xlsx`：4 列（基金全称/原银行间ID/新银行间ID/匹配类型）× 15 行，同色系；
3. `精确匹配说明.md`：含统计四项、有变动明细 5 行、未匹配归类 6 行。

## 4. 理论预期 → 实际结果核对记录

| 核对项 | 理论预期 | 实际结果 | 结论 |
|--------|----------|----------|------|
| 统计四项 | 15/5/4/6 | 引擎输出 15/5/4/6 | ✅ |
| 数值ID判等（金丝雀） | 4 行无变化 | 实际 4 行无变化（1001-1004 判等成功） | ✅ |
| 空值→新ID | 2 行（平衡7/8号）有变动 | 一致 | ✅ |
| 更新表银行间ID 列 | 见 3.1 | 逐格一致 | ✅ |
| 颜色分布 | 行2-5蓝/6-10绿/11-16红 | openpyxl 抽样一致 | ✅ |
| 未匹配归类 | 见 3.3 | md 内容一致 | ✅ |

结论：引擎实际输出与人工理论推导**完全一致**，预期结果可信，予以固化。

## 5. 运行方式

```bash
# 工作目录 server/，使用项目 venv
.venv/Scripts/python.exe tests/golden/m3/make_samples_m3.py    # 1. 构造样本
.venv/Scripts/python.exe tests/golden/m3/build_expected_m3.py  # 2. 重建预期（仅基线变更时）
.venv/Scripts/python.exe tests/golden/m3/test_golden_m3.py     # 3. 回归测试
```

## 6. 回归比对项

1. 统计 dict 与 `expected/expected_stats.json` 逐项相等；
2. 明细 DataFrame 与 `expected/expected_detail_df.pkl` 逐行逐列值相等；
3. 更新表 `银行间ID` 列值与预期 Excel 逐格一致；
4. 三类颜色抽样（更新表 + 明细表：蓝/绿/红代表行）逐格一致；
5. 说明 md 关键内容（统计数字、三类归类提示、有变动明细行数）。
