# M1 黄金样本回归测试说明

## 1. 目的

以**全脱敏虚构样本**为输入，对 M1 基金资产与净值核对引擎（核心为验收基准
`FundAssetReconciler` v1.0）建立黄金样本回归基线：

- 任何引擎封装层、规则加载层、环境依赖（pandas/openpyxl 版本）的变更，
  不得改变基准核对逻辑对相同样本的输出；
- 预期结果先由**人工按 11 步流水线理论推导**（本文件第 3、4 节），
  再由引擎实际运行固化，防止"将错就错"。

## 2. 文件清单

| 文件 | 说明 |
|------|------|
| `make_samples.py` | 样本构造脚本（全脱敏虚构数据），输出到 `samples/golden/` |
| `build_expected.py` | 预期结果生成脚本：引擎跑样本 → `expected/` |
| `test_golden.py` | 回归测试（纯脚本）：重跑引擎并三方比对（统计/数据/颜色）+ 输入校验单测 |
| `expected/核对结果_预期.xlsx` | 预期结果 Excel（含颜色标注） |
| `expected/expected_stats.json` | 预期统计 dict |
| `expected/expected_result_df.pkl` | 预期结果 DataFrame（pickle，保 dtype） |

样本输入：`samples/golden/基金资产表_样本.xlsx`（28 列 × 15 行）、
`samples/golden/净值查询表_样本.xlsx`（9 列 × 17 行）。

## 3. 样本场景 → 理论预期对照表（人工推导）

样本数据全部为虚构脱敏数据。基金资产表 15 行（ZH001~ZH015），
净值查询表 17 行。以下按 11 步流水线逐步推导。

### 3.1 流水线中间过程理论预期

| 步骤 | 理论预期 |
|------|----------|
| 3. 重命名映射 | 净值表 1 条映射：`7001`（测试资本金）→ `6001` |
| 4. 映射去重 | `6001` 组存在原始记录（测试公司资金，总资产 5,000 万）与映射记录（测试资本金，8,000 万）→ **删除原始、保留映射**；净值表剩 16 行 |
| 5. I9 过滤 | `C1004` 组一含 I9 一不含 → 删除"测试稳健A04-I9"；`C1005` 组两条全含 I9 → **全部保留**；净值表剩 15 行 |
| 6. 精确匹配 | 基金 15 行 × 净值 15 行；`C1005` 命中 2 条净值记录 → 基金行复制为 2 行；结果 16 行：精确 14、未匹配 2（C1008、C1009） |
| 7. 模糊匹配 | 未匹配 2 行对可用池（仅 C8008"测试稳益11号A类"）："测试稳益11号" vs "测试稳益11号A类" 相似度 = 2×7/(7+9) = **0.875** ≥ 0.5 → 模糊匹配；"测试孤儿A09" 无候选（且与 C8008 相似度 0.25 < 0.5）→ 未匹配 |
| 8~10. 差异/排序/大宗 | 未匹配（ZH009）置底；其余按最大差异百分比降序；AZ0206、AZ0205 标大宗 |

### 3.3 预期统计 dict

```json
{
    "总记录数": 16,
    "精确匹配": 14,
    "模糊匹配": 1,
    "未匹配": 1,
    "大宗产品数": 2,
    "差异>1.0%数量（非大宗）": 1

```

推导：总记录数 = 15 基金行 + C1005 复制 1 行 = 16；精确 = 16 - 1 模糊 - 1 未匹配 = 14；
大宗 = AZ0206 + AZ0205 = 2；差异>1% 且非大宗仅 ZH002（2.0725%）= 1。

## 4. 理论预期 → 实际结果核对记录

| 核对项 | 理论预期 | 实际结果 | 结论 |
|--------|----------|----------|------|
| 统计 dict 六项 | 见 3.3 | 见 `expected/expected_stats.json`，与理论一致 | ✅ |
| 结果行数/顺序 | 16 行，顺序见 3.2 | 引擎输出与 3.2 逐行一致 | ✅ |
| C1005 一对二复制 | 第 4、16 行 | 一致（分别匹配"I9份额""I9B类"） | ✅ |
| 模糊匹配行字段 | 相似度 0.875、产品名称"测试稳益11号A类"、代码 C1008 | 一致 | ✅ |
| 金丝雀场景（去重/I9 失效检测） | ZH003 差异 0.375%、ZH004 差异 0.3361% | 一致，未出现失效时的巨额差异 | ✅ |
| 颜色分布 | 见 3.4 | openpyxl 逐格读取核对一致 | ✅ |

结论：引擎实际输出与人工理论推导**完全一致**，预期结果可信，予以固化。

## 5. 运行方式

```bash
# 1. 构造样本（已有样本可跳过）
python server/tests/golden/make_samples.py

# 2. 重建预期结果（仅当基准逻辑或样本有经评审的变更时）
python server/tests/golden/build_expected.py

# 3. 回归测试
python server/tests/golden/test_golden.py
```

> Windows Git Bash 下如遇中文输出乱码，前置 `PYTHONIOENCODING=utf-8`。

## 6. 回归比对项

1. **统计 dict**：与 `expected/expected_stats.json` 逐项相等；
2. **结果 DataFrame**：与 `expected/expected_result_df.pkl` 逐行逐列值相等
   （金额等浮点列容差 1e-6，NaN 与 NaN 视为相等）；
3. **输出 Excel 填充色**：与 `expected/核对结果_预期.xlsx` 逐格比对
   `fill.start_color.rgb`（表头至末行 × 15 列全范围）。

另含输入校验单测：列数不足抛 ValueError（提示含"选反"引导语）、
文件选反记录告警日志。
