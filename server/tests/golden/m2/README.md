# M2 黄金样本回归测试说明（基金估值价格核对）

## 1. 目的

以**全脱敏虚构样本**为输入，对 M2 引擎（`server/app/engines/m2_valuation_price.py`）
建立回归基线：先人工按业务规格推导理论预期（本文件第 3 节），再由引擎实际运行
固化，防止"将错就错"。全部产品、证券、价格均为虚构，不含任何真实生产数据。

## 2. 文件清单

| 文件 | 说明 |
|------|------|
| `make_samples_m2.py` | 样本构造脚本（全脱敏虚构数据） |
| `build_expected_m2.py` | 预期结果固化脚本（stats + 逐行明细 + 实际报告副本） |
| `test_golden_m2.py` | 回归测试（纯脚本） |
| `samples/新综合信息查询_基金证券-6301.xlsx` | 系统端 6301：8 行（含 1 零持仓、1 汇总行） |
| `samples/证券投资基金估值表_6301-20260720.xlsx` | 估值表 6301：3 行标题 + 1101/1501/1102 科目层级 |
| `samples/新综合信息查询_基金证券-6302.xlsx` | 系统端 6302：2 行 |

| `expected/` | `expected_stats.json` + `expected_rows.json`（逐行判定） + 报告副本 |

## 3. 样本场景 → 理论预期对照表（人工推导）

引擎配置（黄金回归与 API 冒烟共用口径）：
科目取价规则 = 1101→市价（交易性金融资产）、1501→单位成本（债权投资，
口径提示"摊余成本与市场估值口径差异，属正常"）；fuzzy_sim=0.5；price_tol=0.0001。

### 3.3 合计与热生效预埋

**预期合计**：`{"总记录": 9, "一致": 3, "差异": 3, "单边": 3}`

**配置热生效专项**（test_api_m2.py 执行，不在本回归内）：
新增科目取价规则 1102→市价 后重跑同一任务，估值表 6301 的 1102 叶子
（110109 测试逆回购I，数量 700，市价 100.0500）被提取且系统端无对应 →
6301 统计变为 `{"总记录": 8, "一致": 2, "差异": 2, "单边": 4}`，
报告新增一行科目类型为 "1102-其他投资（取市价）" 的橙行；
恢复（删除该规则）后统计回归基线。

## 4. 防回归金丝雀

1. **零持仓/汇总行过滤**：6301 系统端 8 行仅 6 行有效（第 8、9 行不进报告）；
2. **科目层级过滤**：估值表 1101/1501 汇总行（数量为空）不提取，仅叶子节点进报告；
3. **取价字段区分**：110103 所在 1501 行市价 102.5000、单位成本 100.0000，
   按规则必须取**单位成本**（差异 2.0），若误取市价则差异 0.5 且备注缺失——双断言锁定；
4. **模糊匹配不误配**：未匹配池中的"专项转债E"与"测试存单H"相似度 0.000、
   与"测试国开债F"相似度 0.182，均 < 0.5，必须各自保持单边；
5. **1102 默认不提取**：默认规则下 110109 不进报告（热生效专项的对照组）。
