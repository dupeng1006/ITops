| 文档信息 | |
|---|---|
| 文档版本 | V1.0 |
| 编制日期 | 2026-07-21 |
| 对应平台版本 | v0.6.2 |

## 一、数据库类型总览

平台共使用 **2 个 SQLite 数据库文件** + 可选的 **5 类外部业务库只读直连**。

| 类别 | 库 / 类型 | 用途 | 存放位置 | 写入方 |
|---|---|---|---|---|
| **平台业务库** | SQLite（WAL 模式） | 平台全部业务数据：用户/任务/规则/数据源/调度/审计/收藏 | `server/data/o32ops.db`（部署态 `data/o32ops.db`） | 平台读写 |
| **数据字典库** | SQLite | O32 系统 PDM 物理模型导入成果：模型/表/字段/引用 | `server/data/dictionary.db`（部署态 `data/dictionary.db`） | 仅 PDM 导入时写，查询只读 |
| **外部业务库** | Oracle / MariaDB / MySQL / SQL Server / PostgreSQL | M1/M2 核对直连取数、字典生成 SQL 的目标库 | 客户内网，平台仅配置连接信息 | **严格只读**（四道防线） |

说明：
- 平台库与字典库**相互独立**（两个文件），便于字典库单独随包预置与覆盖更新。
- 平台库默认内嵌 SQLite，用户规模限定 5 人以内，采用低并发设计（WAL 提升读写并发）。
- 外部库通过 `ds_connection` 配置连接，平台侧强制只读（SQL 白名单 + 超时/行数限制 + 只读事务）。

## 二、平台业务库（o32ops.db）表结构

共 **16 张表**。其中 14 张由 SQLAlchemy 实体定义（`entities.py`）建库，
另 2 张由框架/机制自动创建（`apscheduler_jobs`、`schema_version`）。

按职责分为 7 组：

| 分组 | 表 |
|---|---|
| 用户与审计 | sys_user、sys_audit_log |
| 核对任务 | recon_job、recon_result_file、recon_job_item |
| 任务调度 | schedule_job、apscheduler_jobs、sys_config |
| 数据源管理 | ds_connection、ds_query_template |
| 数据字典收藏 | dict_favorite |
| 系统配置 | sys_subject_price_rule |
| 规则配置 | rule_code_mapping、rule_bulk_product、rule_threshold |
| 迁移登记 | schema_version |

### 2.1 用户与审计

| status | VARCHAR(10) | NOT NULL, 默认 active | 状态：active/disabled |
| source | VARCHAR(10) | NOT NULL, 默认 local | 来源：local/ad |
| must_change_password | BOOLEAN | NOT NULL, 默认 1 | 首次登录强制改密标志 |
| created_at | DATETIME | NOT NULL | 创建时间 |

#### sys_audit_log（审计日志表）

```
                                ┌──────────────┐
                                │   sys_user   │ 用户
                                └──────┬───────┘
                                       │ 1
                                       │ (dict_favorite.user_id)

                          ┌────────────▼───────────┐
                          │    dict_favorite       │ 字典表收藏
                          │  table_id ─────────────┼──┐ (跨库逻辑关联, 无外键)
                          └────────────────────────┘  │
                                                      ▼
                                            [dictionary.db].dict_table

  ┌──────────────┐                ┌──────────────────────┐
  │ ds_connection │ 1            N │  ds_query_template   │ 查询模板
  │ (数据源)      │◄───────────────┤  ds_id               │

                                              │ 1
                ┌─────────────────────────────┼───────────────┐

       ┌────────▼─────────┐                   │
       │  schedule_job     │◄──────────────────┘
       │ (定时任务配置)     │
       └────────┬─────────┘
                │ 1 (last_job_id)                ┌──────────────────┐

       ┌─────────────────┐                       └──────────────────┘
       │   recon_job     │ 核对任务实例
       └────────┬─────────┘
                │ 1
     ┌──────────┼──────────────────┐
     │ N (job_id)                  │ N (job_id)
┌────▼──────────────┐      ┌───────▼───────────┐
│ recon_result_file │      │  recon_job_item   │ (M1 产品级明细)
```

独立/弱关联表（无外键，按 param_key 或 product_code 逻辑引用）：
- `sys_audit_log`：独立于所有业务表，按 username/object_id 记录操作
- `sys_config` / `rule_threshold`：键值参数表，独立
- `sys_subject_price_rule`：M2 取价规则，按 subject_prefix 独立配置
- `rule_code_mapping` / `rule_bulk_product`：M1 规则，按 product_code 在引擎运行时逻辑引用
- `schema_version`：迁移登记，独立

## 四、数据字典库（dictionary.db）表结构

承载 O32 系统 PDM 物理模型导入成果，供【数据字典查询】功能只读使用。
规模基线：**20 个模型 / 4,083 表 / 84,176 字段 / 58 条引用关系**
（按表代码跨模型去重后 4,035 表 / 83,958 字段）。

### 4.1 dict_model（PDM 模型表）

| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| id | INTEGER | PK, 自增 | 模型ID |
| file_name | TEXT | UNIQUE, NOT NULL | PDM 文件名 |
| model_name | TEXT | NOT NULL | 模型名（PDM 内 a:Name） |
| biz_group | TEXT | NOT NULL | 业务组（从文件名提取） |
| table_count | INTEGER | NOT NULL | 表数 |

| — | — | UNIQUE (model_id, table_code) | 同模型内表代码唯一 |

### 4.3 dict_column（字典字段表）

| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| id | INTEGER | PK, 自增 | 字段ID |
| table_id | INTEGER | NOT NULL, **FK → dict_table.id** | 所属表 |
| col_code | TEXT | NOT NULL | 字段代码 |
| col_name | TEXT | NULL | 中文字段名 |
| data_type | TEXT | NULL | 数据类型（如 VARCHAR2(30)） |
| is_pk | INTEGER | NOT NULL, 默认 0 | 是否主键 |
| seq | INTEGER | NOT NULL | 字段序号（PDM 内出现顺序，1 起） |
| comment | TEXT | NULL | 注释 |

### 4.4 dict_reference（表引用关系表）

| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| id | INTEGER | PK, 自增 | 引用ID |
| model_id | INTEGER | NOT NULL, **FK → dict_model.id** | 所属模型 |
| ref_name | TEXT | NULL | 关联名（如 ZRT_FK_1_USER） |
| parent_table_id | INTEGER | NOT NULL, **FK → dict_table.id** | 父表 |
| child_table_id | INTEGER | NOT NULL, **FK → dict_table.id** | 子表 |
| joins_json | TEXT | NOT NULL | JOIN 字段对 JSON：[{"parent_col":"...","child_col":"..."}] |

### 4.5 字典库表间关系

```
┌─────────────┐ 1         N ┌─────────────┐ 1         N ┌──────────────┐
│  dict_model │◄────────────┤  dict_table │◄────────────┤ dict_column  │
└──────┬──────┘  model_id   └──────┬──────┘  table_id   └──────────────┘
       │                           │
       │ 1                         │ 1 (parent_table_id / child_table_id)
       │                    N      │
       │              ┌────────────▼───────┐
       └─────────────►│  dict_reference    │  (parent=被引用父表, child=引用子表)
          model_id  N └────────────────────┘
```

字典库与平台库的关联：
- `dictionary.db.dict_table.id` ←（跨库逻辑关联，无外键）— `o32ops.db.dict_favorite.table_id`
- dict_favorite 中 table_code/table_name 冗余缓存，避免每次跨库回查字典库

## 五、外部业务库（只读直连）

平台通过数据源管理连接的外部库**仅用于只读取数**，不在平台侧建表。

| 支持类型 | 驱动 | 用途 |
|---|---|---|
| Oracle | oracledb（免安装） | O32 生产库、估值系统库直连取数 |
| MariaDB | pymysql | 同上 |
| MySQL | pymysql | 同上 |
| SQL Server | pymssql | 同上 |
| PostgreSQL | psycopg2-binary | 同上 |

只读四道防线：① 数据库侧只读账号；② SQL 白名单（sqlparse token 级，仅单条
SELECT/WITH，保存与执行双校验）；③ 执行保护（超时 60s、最大 100 万行、只读事务）；
④ 凭据 Fernet 加密 + 全量审计。

外部库表结构不在本文档范围，由平台【数据字典查询】功能承载（PDM 导入成果见 §4）。
