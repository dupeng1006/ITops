# O32 日常运维平台 —— 操作手册（二期 v0.5.0）

> 适用对象：日常操作人员（operator）、管理员（admin）、只读人员（viewer）。
> 访问方式：浏览器（Chrome / Edge）打开 `http://<服务器>:8000`，
> 进入 Web 前端页面操作（接口文档页 `/docs` 仅供联调参考）。

## 1. 角色与权限

| 角色 | 可见菜单 | 权限 |
|---|---|---|
| admin | 全部 | 核对任务、下载、规则配置、用户维护 |
| operator | 工作台 / 数据核对中心 / 报告归档中心 | 执行核对任务、下载结果、查看规则摘要 |
| viewer | 工作台 / 报告归档中心 | 仅查询历史任务与统计 |

## 2. 登录与改密

1. 打开平台地址进入**登录页**，输入用户名/密码登录；
2. 首次登录（或密码被管理员重置后）弹出**强制改密框**：
   输入新密码（≥8 位）并确认，系统改密后自动重新登录进入工作台；
3. 右上角用户菜单 → **退出登录**；登录态 8 小时有效，过期自动回登录页。

## 3. 工作台

首页展示**今日任务状态卡片**（今日 M1/M3 任务数、进行中、失败数）与
**最近 5 条任务列表**（模块/业务日期/状态/统计摘要/创建人/时间），
点击"全部历史"跳转报告归档中心。

v0.5.0 起，operator 及以上角色在工作台同时可见**统计看板**（详见 §14）：
任务健康度四卡片、M1 差异趋势折线图、持续差异产品表与最近定时执行列表；
viewer 角色仅见任务卡片与最近任务列表。

## 4. 执行 M1 基金资产与净值核对

菜单：**数据核对中心 → 基金资产与净值核对**

1. 页面顶部显示**当前生效规则摘要**（映射条数/大宗个数/两项阈值）；
2. 选择**取数模式**（v0.5.0 起，默认"文件上传"）：
   - **文件上传**：分别拖入（或点选）**基金资产表**与**净值查询表**（.xls/.xlsx），
     选择业务日期（默认当天）；
   - **数据库查询**：分别选择**基金资产模板**与**净值查询模板**
     （下拉按模块过滤并显示所属数据源），**业务日期必选**（作为模板 `:biz_date`
     参数注入查询）；提交前页面回显所选模板与 SQL 摘要供复核
     （数据库查询模式详见 §12）；
3. 点击 **开始核对**；
   - 文件选反或列数不足时，页面直接提示后端中文错误（含"是否文件选反"引导）；
4. 执行中页面显示**进度条 + 实时日志**（2 秒刷新，文案与核对小程序一致）；
5. 完成后展示**统计卡片**（总记录/精确/模糊/未匹配/大宗/差异>1%），
   点击 **下载结果 Excel**（橙=大宗、浅红=差异超阈值、浅黄=未匹配）；
   失败时原样展示中文错误明细，可点"重新上传"。

## 5. 执行 M2 基金估值价格核对（v0.4.0 起）

菜单：**数据核对中心 → 估值价格核对**

1. 选择**取数模式**（v0.5.0 起，默认"文件上传"）：
   - **文件上传**：分别拖入（或点选）**系统端文件**（新综合信息查询_基金证券，.xls/.xlsx，可多选）与
     **估值表文件**（证券投资基金估值表，.xls/.xlsx，可多选）；
     文件名须含**产品标识**（4 位数字，如 `新综合信息查询_基金证券-6301.xlsx`、
     `证券投资基金估值表_6301-20260720.xlsx`），页面实时展示**配对预览**
     （产品标识 / 两侧文件 / 配对状态），存在落单或未识别标识的文件时无法提交；
   - **数据库查询**：按**产品组**选择系统端模板与估值表模板（可多组，
     产品标识可留空由系统端模板名自动派生），**业务日期必选**，
     提交前回显各组模板与数据源供复核（详见 §12）；
2. 选择业务日期（文件模式默认当天），点击 **开始核对**；
   系统端缺关键列（证券代码/证券名称/持仓/估值价格）、估值表缺列或缺取价字段列时，
   页面直接提示后端中文错误（指明缺失列名）；
3. 执行中页面显示**进度条 + 实时日志**（2 秒刷新）；
4. 完成后按**产品 Tab** 展示统计卡片（总记录/一致/差异/单边）与合计，
   每个产品一个 **下载按钮**（`{产品}_估值价格核对报告.xlsx`；
   绿=一致、红=差异、橙=单边）；报告含底部汇总行与备注列
   （1501 差异自动带出口径提示；888880 新标准券单边自动带出说明）。

## 6. 执行 M3 银行间ID匹配

菜单：**数据核对中心 → 银行间ID匹配**

操作与 M1 同构：上传**基金属性表**（.xls/.xlsx）与
**交易成员基本信息表**（GBK 编码 .csv）→ 开始匹配 → 结果页展示统计卡片
（总记录/有变动/无变化/未匹配）与三个下载按钮：
**更新表**（蓝=有变动、绿=无变化、红=未匹配）、**结果明细**、**匹配说明**（未匹配三类归因）。

## 7. 报告归档中心

菜单：**报告归档中心**

- 顶部按**模块 / 业务日期区间 / 状态**筛选，表格分页展示历史任务；
- 行内 **详情**：弹窗查看统计摘要、结果文件与运行日志尾部；
- 行内 **下载**（operator 及以上）：下载该任务结果文件；
- 同日重复执行的结果自动 `_v2`、`_v3` 编号，历史不被覆盖。

## 8. 规则配置中心（仅 admin）

菜单：**规则配置中心**。规则**热生效**：保存后新任务立即按新规则执行，
历史结果不受影响；全部修改记录审计日志（谁、何时、改了什么）。

- **映射规则**：表格展示（原代码→映射后代码），支持新增、编辑、启停开关、删除；
- **大宗产品清单**：同构维护（产品代码 + 说明）；
- **阈值参数**：修改 diff_pct（0.01~100）、fuzzy_sim（0~1）、price_tol（0~1，预留），
  越界提交将被服务端拒绝并提示合法范围；
- **导出配置**：下载与核对小程序同构的 JSON（可直接给小程序使用）；
- **导入配置**：选择小程序 `fund_reconciler_config.json` → 解析预览条数 →
  确认弹窗提示**整体替换**映射与大宗后执行（非法输入整体拒绝，不落半截）。

## 9. 数据源管理（仅 admin，v0.3.0 起）

菜单：**数据源管理 → 连接配置 / 查询模板**。数据库只读直连的配置入口
（M1/M2 任务直连取数在后续版本开放，本版本先完成连接与模板配置）。

- **连接配置**：名称、数据库类型（Oracle/MariaDB/MySQL/SQL Server/PostgreSQL）、
  主机、端口（留空按类型默认）、库名（Oracle 为服务名或 SID 二选一）、账号、密码。
  密码加密存储、页面只显示掩码；编辑时密码留空表示不修改。
  **测试连接**按钮即时验证并弹出中文结果（成功耗时 / 失败原因）；
- **查询模板**：所属模块（M1 基金资产表 / M1 净值查询表 / M2 新综合信息查询 /
  M2 估值表 / M3 交易成员信息）、关联数据源、查询 SQL（仅允许单条 SELECT/WITH，
  保存时安全校验）、字段映射（结果列→标准逻辑字段，大小写不敏感）、
  参数定义（如业务日期 `biz_date`，类型 date/string/integer/number，可设必填）。
  **预览**按填写的参数实际执行查询，返回前 50 行并显示本次生效的执行保护；
- 安全约束：SQL 白名单（保存与执行双校验）、语句超时 60s、最大返回 100 万行、
  只读事务模式；全部配置变更与预览执行留痕审计。数据库侧只读账号授权示例
  见《部署手册》第 9 章。

## 10. 系统配置（仅 admin，v0.4.0 起）

菜单：**系统管理 → 系统配置**。维护 M2 估值价格核对的**科目取价规则**；
规则**热生效**：保存后新建任务立即按新规则执行，历史任务归档结果不受影响；
全部新增/修改/删除/启停留痕审计。

- 列表展示：科目前缀、取价字段、科目说明、口径提示语、排序号、启停开关、修改人/时间；
- **新增/编辑**：科目前缀（如 1101/1501，全库唯一）+ 取价字段
  （估值表列名，预置 市价/单位成本/成本，可自定义输入）+ 科目说明 +
  口径提示语（差异行「备注」列自动带出，可空）+ 排序号（小者先提取）+ 启停；
- 初始预置两条：`1101→市价（交易性金融资产）`、`1501→单位成本（债权投资，
  口径提示：摊余成本与市场估值口径差异，属正常）`；
- 注意：取价字段必须与估值表实际列名一致，否则建任务时校验报错并指明缺失列名。

## 11. 用户维护（仅 admin）

菜单：**系统管理 → 用户维护**

- **新增用户**：用户名 + 初始密码（≥8 位）+ 角色；
- **编辑**：调整角色、启用/停用（初始 admin 不可停用/降权）；
- **重置密码**：重置后该用户下次登录强制改密；
- **删除**：初始 admin 不可删除。

## 12. 数据库查询模式（v0.5.0 起）

M1/M2 建任务支持两种**取数模式**：`文件上传`（默认，既有行为不变）与
`数据库查询`。数据库查询模式的链路为：

1. **查询模板**（菜单：数据源管理 → 查询模板，仅 admin 维护）：
   模板绑定数据源，SQL 仅允许单条 SELECT/WITH（保存时经 SQL Guard 白名单校验），
   声明 `:biz_date` 参数；任务创建时平台以业务日期绑定参数执行查询；
2. **同步取数**：建任务时立即查询（不先入队），查询失败直接返回中文错误
   （模板不存在 404 / 模板与模块不匹配 400 / 缺业务日期 400），不产生半截任务；
3. **查询快照归档**：查询结果以 UTF-8-SIG CSV 落入
   `archive\{模块}\{业务日期}\{任务ID}\input\查询快照_*.csv`（M1 另有引擎消费件 xlsx），
   可回查、可重算比对；
4. **同一核对引擎**：快照落盘后走与文件模式完全相同的校验与核对流水线，
   统计口径与结果 Excel 版式完全一致（已由黄金样本一致性测试验证）。

适用场景：每日数据就绪后免导出直接核对；与 §13 任务调度中心配合可实现全自动化。

## 13. 任务调度中心（v0.5.0 起）

菜单：**任务调度中心**（operator 及以上；APScheduler 持久化调度，服务重启不丢任务）

- **新建定时任务**：填任务名称、模块（M1/M2）、取数模式与 cron 表达式
  （5 段：分 时 日 月 周；提供 17:47/18:07/18:23/18:37 等常用快捷选项，
  建议避开整点/半点；非法表达式提交即报中文错误）；
  - db 模式：选择配置好的查询模板（M1 两个模板；M2 按产品组选模板）；
  - file 模式：填**监测目录**（绝对路径）。到点执行时检查一次目录
    （不做常驻监测）：M1 取文件名含「基金资产」「净值」的最新文件各一，
    M2 取含「新综合信息查询」「估值表」的文件按产品标识配对；
    文件未就绪则任务标记**待文件**，并在重试间隔后自动再试一次；
- **列表操作**：启停开关、编辑、删除、**立即执行**（不等 cron 到点，
  手动触发一次完整流程）、**执行历史**抽屉（复用核对任务列表，
  按 trigger_type=schedule 过滤）；
- **失败重试**：执行失败（含取数失败、文件未就绪）隔
  `schedule_retry_delay_minutes`（默认 5 分钟）自动重试 1 次，
  仍失败置"失败"并记录原因（列表可见最近失败提示）；
- **就绪时间参数**：`data_ready_time`（默认 17:30）与 `buffer_minutes`
  （默认 30）在 系统管理 → 系统配置 → 系统参数 中维护（admin），
  用于 file 模式就绪判定与提示文案；
- 定时自动任务**不预置**，全部由用户按需创建；全部操作留痕审计。

## 14. 统计看板与工作台（v0.5.0 起）

工作台（operator 及以上）在看板区展示：

1. **任务健康度四卡片**：定时任务启用/总数、近 30 天定时执行成功数、
   失败数、"待文件"数（含就绪时间参数回显）；
2. **M1 核对差异趋势**（ECharts 折线图，7/30/90 天切换）：
   按业务日期展示总记录/精确匹配/未匹配/差异>1%（非大宗）四条曲线，
   同一业务日期取当日最后一次成功任务；
3. **持续差异产品**：近 7 日差异超阈值（rule_threshold.diff_pct，默认 1.0%）
   出现 ≥2 次的产品清单（产品代码/名称/出现次数/最近差异%/最大差异%），
   数据来自 M1 任务成功时自动入库的产品级明细（recon_job_item，
   历史任务不回填）；
4. **最近定时执行**：最近 10 次定时任务执行结果与失败原因，
   一键跳转任务调度中心。

viewer 角色工作台不显示看板区（看板接口 operator 及以上）。

## 15. 数据字典查询（v0.5.1 起，operator 及以上）

菜单【数据字典查询】提供 O32 全模型表结构检索与自助查询 SQL 生成。

1. **浏览与搜索**：左栏按业务组/模型过滤，搜索框支持表代码、中文表名、
   字段代码、字段中文名模糊匹配（结果标注命中点）；表行星标可收藏常用表
   （收藏存本机浏览器，不同机器不共享）。
2. **表结构查看**：点击表展示全字段（字段代码/中文名/数据类型/主键 PK 标识/
   注释）与关联表清单（父表/子表两向，点击可跳转）。
3. **生成 SELECT（Oracle 方言）**：勾选字段（或"全选字段加入"）→ 添加条件
   （=,!=,>,>=,<,<=,LIKE,IN,BETWEEN,IS NULL/IS NOT NULL；IN 逗号分隔多值，
   BETWEEN 两个值逗号分隔；DATE 列日期值自动生成 TO_DATE）→ 点击"生成 SQL"。
   默认附加 `ROWNUM <= 500`（可关，可改上限）。
4. **多表自动 JOIN**：选择多张表时按 PDM 外键关系自动推导 JOIN（提示
   "已按外键 xxx 自动 JOIN yyy"，路径可经中间表桥接）；无已知外键关联的表
   按 CROSS JOIN 输出并显示警告"无已知外键关联，请自行确认 JOIN 条件"。
5. **复制与保存模板**：生成 SQL 一键复制；admin 可"保存为查询模板"
   （模块 custom，选执行数据源），保存后在 数据源管理→查询模板 中统一维护。

安全约束：表/列/别名全部来自字典库白名单，条件值自动转义，生成 SQL 经
只读校验（SQL Guard）自证；字典库为独立 SQLite（data\dictionary.db，
由部署方预置），模型更新后由管理员重跑 `server\scripts\import_pdm.py`
重建（PDM 文件不进部署包）。

## 16. 常见问题

| 现象 | 处理 |
|---|---|
| 登录后自动回登录页 | 登录态过期（8 小时），重新登录 |
| 上传后提示"列数不足/是否文件选反" | 按提示核对两个文件是否选反或模板是否正确 |
| 结果统计与预期不符 | 先核对规则配置（规则热生效，最近变更见审计）；再用归档中心对比历史结果 |
| M2 提示"未能配对估值表/系统端文件" | 检查文件名产品标识（4 位数字）是否两侧一致，落单文件补齐后重新提交 |
| M2 提示"缺少科目取价规则所需的取价字段列" | 到 系统管理→系统配置 核对取价字段与估值表列名是否一致 |
| M2 报告出现橙色单边行 | 先查备注列（888880 等已知过程科目有说明）；其余单边需业务侧确认是否当日新券或赎回 |
| 无法看到规则配置/用户维护菜单 | 当前角色非 admin，联系管理员 |
