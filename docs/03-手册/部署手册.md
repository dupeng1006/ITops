# O32 日常运维平台 —— 部署手册（二期 v0.3.0）

> 适用对象：运维实施人员。目标环境：**无任何 Python/Node 等运行时的
> Windows 服务器**（Windows Server 2016+ / Windows 10+，x64），解压即用。

## 1. 交付物

| 项 | 说明 |
|---|---|
| `o32-ops-platform-v0.3.0.zip` | 一体化部署包（约 50 MB，解压后约 113 MB） |

部署包目录结构：

```
o32-ops-platform-v0.3.0\
├── app\                # 服务端程序（o32-server.exe + _internal 运行时依赖）
├── data\               # 平台数据（SQLite 库 + 密钥，首启自动建库，升级不覆盖）
├── archive\            # 核对归档（上传原件与结果文件，升级不覆盖）
├── logs\               # 运行日志（server.log）
├── nssm\               # NSSM 2.24 服务化工具（可选，含来源与 SHA-256 说明）
├── install.bat         # 注册开机自启（需管理员；NSSM 优先，失败回退计划任务）
├── uninstall.bat       # 卸载自启（需管理员）
├── start.bat           # 免服务前台启动（试运行用）
├── stop.bat            # 停止前台启动的实例
├── 版本.txt            # 版本号、构建日期、内容清单、变更摘要
└── 依赖清单.txt        # 全部依赖版本与 License 类型
```

## 2. 环境要求

- 操作系统：Windows x64（Server 2016+ 或 Windows 10+），**无需安装任何运行时**；
- 磁盘：≥ 500 MB 可用空间（程序约 111 MB，数据与归档随使用增长）；
- 网络：开放服务端口（默认 **8000**，可用 `--port` 或环境变量 `O32OPS_PORT` 修改）；
- 权限：仅"注册开机自启"需要管理员；试运行（start.bat）普通用户即可。

## 3. 部署步骤

1. 将 zip 解压到目标目录（建议 `D:\O32OpsPlatform\`，路径勿含中文空格更佳）；
2. **试运行**：双击 `start.bat` → 弹出的服务端窗口显示启动横幅
   （服务地址/默认账号/数据目录）；
3. **正式运行（可选）**：右键"以管理员身份运行" `install.bat`
   - 检测到 `nssm\nssm.exe` 时优先注册为 Windows 服务（开机自启）；
   - NSSM 不可用或被策略拦截时自动回退为"计划任务"（SYSTEM 账户、开机自启）；
4. 首次启动自动完成：建库 `data\o32ops.db` → 建表 → 创建初始管理员
   **admin / Admin@123** → 导入初始规则（21 条映射、11 个大宗产品、3 项阈值）。

### 3.1 前端页面访问

部署包 `app\web\` 内置 Web 前端（v0.2.0 起），服务端启动后直接以
**浏览器（Chrome / Edge）访问** `http://<服务器>:8000` 即可使用：

- 登录页输入初始账号 **admin / Admin@123**，按提示强制改密后进入工作台；
- 页面涵盖：工作台、数据核对中心（M1/M3 任务）、报告归档中心、
  规则配置中心（admin）、数据源管理（admin，v0.3.0 起）、系统管理-用户维护（admin）；
- 接口文档页 `http://<服务器>:8000/docs` 仍可用于联调与核对；
- 升级替换 `app\` 目录时前端随之一并更新，无需单独部署。

## 4. 初始化后必做事项

1. 登录 `admin / Admin@123`，**立即修改初始密码**（系统强制首登改密）；
2. 按需要创建 operator / viewer 账号（用户维护接口，仅 admin）；
3. 核对规则条数：`GET /api/rules/mappings` 应为 21 条、
   `GET /api/rules/bulk-products` 应为 11 个、
   `GET /api/rules/thresholds` 应含 diff_pct=1.0 / fuzzy_sim=0.5 / price_tol=0.0001；
4. 用一份近期真实两表跑一次 M1 核对，确认结果与既有小程序一致后再投入日常。

## 5. 启停与回滚

| 操作 | 方式 |
|---|---|
| 启动（试运行） | `start.bat`（前台新窗口，日志可见） |
| 停止（试运行） | `stop.bat` 或关闭服务端窗口 |
| 注册自启 | `install.bat`（管理员） |
| 卸载自启 | `uninstall.bat`（管理员；**data/archive 保留不删**） |
| 升级（推荐：平级目录法） | 停止旧版服务 → 新包解压到旧版**同级目录** → 运行新包内 `upgrade.bat`（自动迁入旧版 data/archive/logs：用户、规则、模板、数据源、历史任务、归档全保留）→ 运行新版 `start.bat`。旧目录原样保留作回滚备份，验证无误后可手动删除 |
| 升级（备选：原地替换法） | 停止服务 → 将旧 `app\` 改名为 `app_旧版本\` → 放入新 `app\` 与新 `*.bat` → 启动（data/archive/logs 不动） |
| 结构升级 | 首次启动时服务端自动完成数据库结构迁移（schema_version 表登记，幂等；启动日志可见），无需人工操作 |
| 回滚 | 停止新版服务 → 回到旧目录运行其 `start.bat`（平级目录法下旧数据未被改动，可直接回退） |
| **禁止** | **不要把新包直接覆盖解压到旧目录，也不要用新包的 `data\` 替换旧数据——否则会清空全部用户、配置与历史数据** |

## 6. 备份建议

- 备份对象：`data\`（平台库+密钥）与 `archive\`（归档）两个目录，
  直接复制即可（建议每日一次，保留最近 30 份）；
- 恢复：停止服务 → 还原目录 → 启动；
- `app\` 无需备份（可从部署包重新解压）。

## 7. 故障排查

| 现象 | 排查 |
|---|---|
| 启动后无法访问 | ① 服务端窗口是否报错；② 端口被占用：`netstat -ano \| findstr :8000`，改用 `--port` 或释放端口；③ 防火墙放行入站 TCP 8000 |
| 登录后提示令牌失效 | `data\secret.key` 被删除会导致历史令牌失效，重新登录即可（请勿删除该文件） |
| 首启无规则数据 | 查看 `logs\server.log` 是否提示 rule_config.json 缺失（正常应随包内置，自动导入） |
| 服务注册失败 | 确认以管理员身份运行 install.bat；NSSM 被拦截时脚本自动回退计划任务，留意窗口提示 |
| 查看日志 | `logs\server.log`（服务运行日志）；每次核对任务的 `run.log` 在 `archive\<模块>\<日期>\<任务ID>\result\` 下 |

## 8. 安全说明

- JWT 密钥首启自动生成并持久化于 `data\secret.key`（仅本机留存），
  也可通过环境变量 `O32OPS_SECRET_KEY` 显式指定；
- 默认监听 `0.0.0.0`（允许局域网访问），仅本机使用可 `o32-server.exe --host 127.0.0.1`；
- 一期为内网 HTTP 部署，跨网段访问请置于反向代理（HTTPS）之后（二期规划）。

## 9. 数据源配置（二期 · 数据库只读直连）

平台支持以"数据库查询"模式取数（M1/M2 任务直连取数在后续版本开放，本版本先交付
数据源连接与查询模板配置能力）。配置入口：管理员登录 → 左侧菜单 **数据源管理**：

- **连接配置**：名称、类型（Oracle / MariaDB / MySQL / SQL Server / PostgreSQL）、
  主机、端口、库名（Oracle 为服务名或 SID 二选一）、账号、密码、连接参数。
  密码以 Fernet 对称加密存储（密钥由 `data\secret.key` 派生），任何接口均不返回明文；
  编辑时密码留空表示不修改。"测试连接"按钮即时验证连通性并返回中文结果；
- **查询模板**：每条数据需求配置为一个模板：所属模块（M1 基金资产表 / M1 净值查询表 /
  M2 新综合信息查询 / M2 估值表 / M3 交易成员信息）、关联数据源、查询 SQL、
  结果列→标准逻辑字段映射、参数定义（如业务日期 `:biz_date`，支持 date/string/
  integer/number 类型与必填约束）。"预览"按参数实际执行查询并返回前 50 行。

### 9.1 只读安全防线（平台侧强制）

| 防线 | 措施 |
|---|---|
| SQL 白名单 | 查询模板保存时与每次执行前各校验一次：仅允许单条 SELECT/WITH，拦截一切写操作、DDL、存储过程调用、多语句、SELECT INTO、行锁（FOR UPDATE / LOCK IN SHARE MODE） |
| 执行保护 | 语句超时（默认 60s，`O32OPS_DS_QUERY_TIMEOUT` 可配）、最大返回行数（默认 100 万，`O32OPS_DS_MAX_ROWS` 可配）、只读事务模式（Oracle/PostgreSQL `SET TRANSACTION READ ONLY`，MySQL/MariaDB `SET SESSION TRANSACTION READ ONLY`，尽力而为并留痕） |
| 凭据与审计 | 密码 Fernet 加密存储；数据源增删改、测试连接、模板变更、每次预览执行全部审计留痕 |

### 9.2 数据库侧只读账号授权示例（防线一，务必执行）

平台侧白名单不能替代数据库侧最小授权。请 DBA 为平台创建**只读账号**，
禁止直接使用业务写账号：

**Oracle**
```sql
CREATE USER o32ops_read IDENTIFIED BY "<强密码>";
GRANT CONNECT TO o32ops_read;
-- 按需对业务表逐个授权（示例为 O32 常用对象，按实际库表调整）
GRANT SELECT ON o32schema.fund_asset_table TO o32ops_read;
GRANT SELECT ON o32schema.netvalue_view  TO o32ops_read;
-- 批量视图（DBA 执行）：SELECT 'GRANT SELECT ON '||owner||'.'||table_name||' TO o32ops_read;'
-- FROM all_tables WHERE owner='O32SCHEMA';
```

**MariaDB / MySQL**
```sql
CREATE USER 'o32ops_read'@'%' IDENTIFIED BY '<强密码>';
GRANT SELECT ON o32db.* TO 'o32ops_read'@'%';   -- 仅 SELECT，禁止其他任何权限
FLUSH PRIVILEGES;
-- 核验：SHOW GRANTS FOR 'o32ops_read'@'%';
```

**SQL Server**
```sql
CREATE LOGIN o32ops_read WITH PASSWORD = '<强密码>';
USE o32db;
CREATE USER o32ops_read FOR LOGIN o32ops_read;
EXEC sp_addrolemember 'db_datareader', 'o32ops_read';   -- 只读角色
-- 显式拒绝写（双保险）：DENY INSERT, UPDATE, DELETE ON SCHEMA::dbo TO o32ops_read;
```

**PostgreSQL**
```sql
CREATE ROLE o32ops_read LOGIN PASSWORD '<强密码>';
GRANT CONNECT ON DATABASE o32db TO o32ops_read;
GRANT USAGE ON SCHEMA public TO o32ops_read;
GRANT SELECT ON ALL TABLES IN SCHEMA public TO o32ops_read;
ALTER DEFAULT PRIVILEGES IN SCHEMA public GRANT SELECT ON TABLES TO o32ops_read;  -- 未来新表
-- 强制只读事务（双保险）：ALTER ROLE o32ops_read SET default_transaction_read_only = on;
```

授权完成后，在平台"数据源管理 → 连接配置"中录入并使用**测试连接**验证。
