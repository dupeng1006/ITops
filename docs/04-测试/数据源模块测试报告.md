## 2. SQL Guard 只读白名单（75 用例）

本模块最重要的安全门禁。规则：恰好一条语句 + 首关键字 SELECT/WITH +
token 级黑名单（INSERT/UPDATE/DELETE/MERGE/REPLACE/UPSERT/CREATE/DROP/ALTER/
TRUNCATE/RENAME/GRANT/REVOKE/CALL/EXEC/EXECUTE/INTO/LOCK）。

### 2.1 合法放行（28 例，全部通过）

简单/小写/大小写混合 SELECT；WHERE/GROUP/ORDER；子查询；JOIN；UNION；
WITH 单/多 CTE；WITH RECURSIVE；行首单行与块注释；行尾分号（+注释）；
前导空白；字符串内含分号/危险词/注释符；别名含危险词形（update_time 等）；
REPLACE 函数；保留字裸列名（comment/start/set）；限定名含保留字；
Oracle hint 注释；`:biz_date` 绑定参数；CAST/窗口函数。

### 2.2 非法拦截（41 例，全部通过，400 中文提示）

| 类别 | 用例 |
|---|---|
| DML 写 | INSERT、INSERT SELECT、UPDATE、DELETE、MERGE、REPLACE 语句、小写 insert 绕过、大小写混合 dRoP |
| DDL | DROP、CREATE TABLE/INDEX、ALTER、TRUNCATE、RENAME |
| DCL | GRANT、REVOKE |
| 存储过程/动态执行 | CALL、EXEC、EXECUTE IMMEDIATE、xp_cmdshell 链（多语句拦截） |
| 多语句 | 查询+写、两条查询、写在前、注释分隔伪装（`/*;*/ ;`）、BEGIN…DELETE…COMMIT |
| 隐蔽写 | SELECT INTO（MSSQL 建表）、WITH 包裹 DELETE/INSERT（PG）、CTE 内 DML RETURNING |
| 锁 | FOR UPDATE、FOR UPDATE NOWAIT、LOCK IN SHARE MODE |
| 语句级 | SET、USE、COMMIT、EXPLAIN（虽是只读，仅 SELECT/WITH 白名单不放行）、SHOW |
| 注释绕过 | 行注释后藏 DROP、块注释后藏 DELETE |
| 空输入 | 空串、纯空白、纯注释 |

### 2.3 误伤防护复核（6 例，全部通过）

危险词出现在字符串字面量（`'insert into log'`、`'x; DROP TABLE y'`）、
引号标识符（`"update"`、`` `delete` ``、`[lock]`）一律不误拦。

设计注记：黑名单仅收录"可在 SELECT 中段造成写/锁"的关键字；语句级危险词
（SET/USE/BEGIN 等）由首关键字白名单拦截，避免裸列名误伤（`SELECT comment`
中 comment 会被 sqlparse 词法识别为 Keyword）。已知边界：列名与保留字完全相同
且未加引号时可能误拦，规范写法为加引号或换别名（已在代码注释与部署手册标注）。

## 3. DbAdapter 集成测试（28 用例，SQLite 方言）

- **基础查询**：20 行样本全量返回；protections 留痕含白名单/行数限制/PRAGMA query_only；
- **字段映射**：大小写不敏感（CODE→product_code、Name→product_name），未映射列保留原名，
  映射计数留痕（3/4 列）；
- **参数绑定**：date 校验（yyyyMMdd 与 yyyy-MM-dd 双格式）；`:biz_date` 过滤精确命中；
  注入式参数（`' OR '1'='1`）被 date 校验拒绝、string 参数按字面绑定（0 行而非全表）——
  绑定参数天然防注入实证；缺必填/未定义参数/非法 integer 均中文报错；
- **行数硬限制**：max_rows=5 查 20 行触发中文超限报错（含上限值）；恰好等于上限放行；
- **预览截断**：limit_rows=5 截断语义正确（不改变超限语义）；
- **纵深防御**：DELETE 经 execute_query 被 SQL Guard 执行前二次校验拦截；
- **友好报错**：不存在主机（MySQL getaddrinfo 失败）→"无法解析数据库主机名"；
  非法库路径→中文报错；报错信息不含密码；
- **URL 构造**：Oracle 缺服务名/SID 报错；service_name/SID 两种形态；端口默认值 1521；
  MySQL 密码特殊字符转义（render_as_string 验证）；不支持类型报错。

## 4. 数据源 API 冒烟（76 用例）

链路：admin/operator/viewer 三账号 → 数据源 CRUD → 测试连接 → 模板 CRUD →
预览 → 权限边界 → 审计核验。重点断言：

- **密码安全**：密文落库（非明文、Fernet 可逆验证）；列表/详情恒返回掩码 `********`；
  留空密码不修改（密文不变）；更新密码密文变化且可解密为新值；审计全文不含密码明文；
- **SQL Guard 保存拦截**：INSERT/DELETE/多语句/SELECT INTO/FOR UPDATE 建模板全部 400；
- **执行前二次校验**：直接改库把模板 SQL 篡改为 DELETE 后调用 preview 仍 400（模拟绕过接口场景）；
- **preview**：参数绑定 + 字段映射列名 + 行内容与样本一致 + protections 留痕；
  缺必填参数/日期格式非法 400；
- **测试连接**：SQLite 成功（中文+耗时）；MySQL 不存在主机 success=false + 中文友好报错；
- **引用约束**：被模板引用的数据源删除 400；
- **权限边界**：operator 可查/测连/预览、写操作 403；viewer 查询 403；未登录 401；
- **审计**：ds_create/ds_update/ds_delete/ds_test/tpl_create/tpl_update/tpl_delete/
  tpl_preview 八类动作全部落库。

## 6. 未覆盖项与遗留（如实标注）

1. **五类生产方言实库连通性未验证**：本机无 Oracle/MariaDB/MySQL/MSSQL/PG 实体库，
   真实库的只读事务设置（SET TRANSACTION READ ONLY 等）、语句超时、驱动行为
   未做实证；URL 构造与报错翻译以单测/冒烟覆盖。建议部署环境按部署手册 §9.2
   创建只读账号后，用"测试连接 + preview"对每种方言各做一次实证并归档结果。
2. **Oracle thick 模式**：thin 模式不支持的旧特性（如 OS 认证、部分高级类型）
   未覆盖；extra 连接参数 JSON 已预留切换位。
3. **MSSQL 无会话级只读事务**：该方言依赖"只读账号 + SQL Guard"两道防线
   （代码注释与部署手册已注明），未做 db_datareader 实库验证。
4. **语句超时为尽力而为**：PostgreSQL/MySQL 有会话级设置；Oracle/MSSQL/SQLite
   无通用会话级语句超时，依赖行数限制与账号侧资源限制兜底。
5. **页面交互未做浏览器自动化**：数据源两页的表单交互（动态字段、预览弹窗）
   经构建与接口级集成验证；登录后页面操作未做浏览器端自动化实测（同前版口径）。
6. **M1/M2 任务 db 取数联动**：本步不做（需求 DS-F4），下一步实现任务创建时
   可选"数据库查询"模式并接入既有核对引擎。
